# Ztocks — Private Tokenized Stock Investing on Stellar

**Stellar Hacks: Real-World ZK Hackathon submission.** Buy synthetic
tokenized stocks (AAPLx, TSLAx, NVDAx, GOOGLx) on Stellar while
proving portfolio solvency via Groth16 zero-knowledge proofs, verified
on-chain in Soroban — without ever revealing individual holdings.

This repo is the frontend. Three sibling pieces complete the system:

| Piece | Location | What it does |
|---|---|---|
| Soroban contracts + ZK circuit | `../ztocks-contracts/` | vault, lending, price-oracle, zk-verifier contracts + the Circom circuit |
| Price pusher | `scripts/price-pusher/` | Pushes real AAPL/TSLA/NVDA/GOOGL quotes on-chain |
| This frontend | (this repo) | Next.js app, wired to live contract reads/writes |

## Full setup order (do not skip steps)

### 1. Deploy contracts + build the ZK circuit
Follow `ztocks-contracts/contracts/README.md` start to finish:
circuit trusted setup → deploy price-oracle → deploy zk-verifier →
deploy vault → deploy lending → fund the lending pool.

### 2. Generate typed contract bindings
```bash
cp .env.example .env.local   # fill in the 5 contract IDs from step 1
chmod +x scripts/generate-bindings.sh
./scripts/generate-bindings.sh
```
This creates `packages/{vault,lending,price-oracle,zk-verifier}` —
real, fully-typed NPM packages generated by the Stellar CLI directly
from each deployed contract's on-chain spec. Add them to
`package.json` as local dependencies, e.g.:
```json
"vault-client": "file:./packages/vault",
"lending-client": "file:./packages/lending",
"price-oracle-client": "file:./packages/price-oracle",
"zk-verifier-client": "file:./packages/zk-verifier"
```
then `npm install` again. Until you do this, the app still **builds
and runs** (see `next.config.ts` — these are configured as optional
webpack externals) but every page shows a clear "Setup needed" banner
instead of silently displaying fake numbers.

### 3. Start the price pusher
```bash
cd scripts/price-pusher
cp .env.example .env   # add your Finnhub/Twelve Data key + pusher secret key
npm install
npm start
```
Without this running, the oracle has no prices and the trade/
portfolio pages will show "—" for asset values rather than invent a
number.

### 4. (Optional but required for real borrowing) Stand up the proof encoder
ZK proof generation runs in-browser via snarkjs, but converting the
proof into the exact bytes Soroban's BLS12-381 verifier expects needs
a tiny backend wrapping `cargo run --bin encode_proof` — see the large
comment block at the top of `hooks/useZkProof.ts` for exactly why this
isn't done in JS directly (a real, documented cross-library byte-
encoding risk, not a shortcut). Point `NEXT_PUBLIC_PROOF_ENCODER_URL`
at it once built.

### 5. Run the frontend
```bash
npm install
npm run dev
```

## What's real vs. illustrative in this codebase

Every page under `/trade`, `/portfolio`, `/lending` reads and writes
live on-chain state once the steps above are complete — no mock
arrays remain in any of those flows. Two sections on the landing page
(`PortfolioPreview`, `LendingSection`) show clearly-labeled **"Sample
Data"** for visitors who haven't connected a wallet yet, since there's
no real portfolio to show someone who isn't logged in; both link
through to the real, live pages. `HowItWorksSection` is static
educational content about how the ZK proof flow works, not financial
data.

## Deploying to Vercel

```bash
vercel --prod
```
Set the same environment variables from `.env.local` in your Vercel
project settings (Settings → Environment Variables) before deploying
— the build succeeds without them, but the live app will show setup
banners until they're set.

## Tech Stack

- **Frontend**: Next.js 15 (App Router, TypeScript), TailwindCSS, Framer Motion
- **Wallet**: Freighter (`@stellar/freighter-api`)
- **Chain**: Stellar Soroban, contract bindings via `@stellar/stellar-sdk`
- **ZK**: Circom (BLS12-381) + snarkjs (browser proving) + Groth16 verifier in Rust
- **Oracle**: Custom pushed-price oracle (Finnhub/Twelve Data → Soroban)
